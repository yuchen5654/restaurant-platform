# Restaurant Platform

Multi-tenant restaurant data & analytics platform for independent operators (1–5 locations, $500K–$3M revenue, owner-operated). Unifies sales, inventory, labor, and customer data that today lives scattered across Toast, MarketMan, 7shifts, and delivery platforms.

## How to work on this project

1. **The build is broken into steps** in `docs/steps/`. Each step file is self-contained.
2. **Before building ANY step, read its step file first** (`docs/steps/step-XX-*.md`), plus `docs/conventions.md`.
3. **Read `docs/architecture.md`** if you need the big-picture "why" behind a decision.
4. **Log any non-obvious decision** you make mid-build in `DECISIONS.md` (with the date and the reasoning).
5. **Update the Build Status checkboxes below** the moment a step is complete.
6. **One step per session.** Do not attempt multiple steps at once — scope each session to a single step file.
7. **Summarize the step before writing code.** When asked to build a step, first restate what it involves and what files you'll create, then proceed.
8. **Verify every step** with a throwaway verification script (seed → exercise → assert → delete the script and its seed data). Frontend steps verify via `npm run build` (0 TS errors) + a visual browser check.

## Hard Rules (never violate)

- **`restaurant_id` on every data model.** Multi-tenant isolation is non-negotiable. Every query must be scoped to the authenticated restaurant.
- **All ingestion follows stage → review → confirm.** Never auto-commit OCR/voice/email/CSV-extracted data straight to live tables. It always lands in `staged_ingestions` first for operator confirmation.
- **Point-in-time data is append-only.** Never overwrite an inventory count, a sale, or an invoice price. Append a new timestamped record.
- **Run `alembic upgrade head` after every schema change.** Never `ALTER TABLE` by hand. **Review autogenerated migrations before applying** — strip any `drop_index` lines touching TimescaleDB hypertable partition indexes (a known recurring artifact).
- **Layer separation:** business logic in `app/services/`, HTTP handlers in `app/routers/`, ORM models in `app/models/`, Pydantic schemas in `app/schemas/`.
- **Negative stock is a signal, not a bug.** Log it visibly — it means a count, invoice, or waste entry is missing. Do not silently clamp and move on.
- **Normalize IDs with `_to_uuid()`** before any comparison — string IDs from URLs/JSON are not equal to UUID objects.

## Known Caveats (added July 2026)

- **Mock-verified integrations:** Toast (Step 5), OCR and voice (Step 5B) were verified against mocked API responses. Before trusting live data: obtain real credentials, print one real response, confirm field names match the parsing code.
- **AI features built but dormant:** `ANTHROPIC_API_KEY` and `OPENAI_API_KEY` are empty in `backend/.env`. The Ask page and OCR/voice work end-to-end the moment real keys are added (pay-per-use API billing — separate from any Claude subscription).
- **Dev login:** `owner@testbistro.com` / `devpassword123` (restaurant "Test Bistro"). This is a dev-only seed.
- **`SECRET_KEY` is a placeholder** — must become a real secret (via secrets manager) before any deployment.
- **Windows dev quirks:** `localhost` vs `127.0.0.1` can resolve differently — frontend API base uses `127.0.0.1:8000`; CORS allows ports 3000 and 5173. CRLF→LF git warnings are harmless.

## Stack (as actually built)

| Layer | Tech |
|-------|------|
| Backend | Python 3.14 + FastAPI (uvicorn on 127.0.0.1:8000) |
| Database | PostgreSQL 15 + TimescaleDB (Docker) |
| Background jobs | Celery + Redis (Docker); beat: Toast pull 3:00am, alerts 3:30am |
| Ingestion (OCR) | Anthropic Claude Vision API |
| Ingestion (voice) | OpenAI Whisper |
| Ingestion (email) | SendGrid Inbound Parse webhook |
| Frontend | React 18 + TypeScript + **Vite** (port 5173) + Tailwind + TanStack Query v5 + Recharts |
| ML (gated) | LightGBM |
| LLM | Anthropic Claude API (`claude-sonnet-4-6`) — structured-context Q&A, NOT RAG |
| Hosting (gated) | AWS (RDS + EC2 + S3 + CloudFront) |

## Local Dev

```bash
docker compose up -d                          # Postgres/TimescaleDB + Redis
cd backend && venv\Scripts\activate           # (Windows) or source venv/bin/activate
uvicorn app.main:app --reload                 # API at http://127.0.0.1:8000
# Interactive API docs: http://127.0.0.1:8000/docs

# Celery (separate terminals):
celery -A app.workers.celery_app worker --loglevel=info
celery -A app.workers.celery_app beat --loglevel=info

# Frontend:
cd frontend && npm run dev                    # http://localhost:5173  (Vite)
```
